Schedulers

A scheduler determines the scheduling policy during a simulation run (using the @mce_alg_simulation@ algorithm). Figure 3 illustrates a particular implementation.

Figure 3: A custom scheduler

The scheduler forbids any action which is the result of sending a message to a process with the registered name @locker@. This scheduler is used in the example on page  to steer a simulation algorithm, which is followed by running a model checking algorithm (@mce_alg_safety@). The idea is that the simulation part should explore the initial startup of the system, forbidding any action that involves actually requesting resources from the @locker@ process. Once there is no schedulable transition (except the ones involving sends to the locker) the simulation phase ends, and model checking begins.
